### 平均每回合成本（含週期性攤銷）

```
攤銷計算：
  階段摘要：$0.000140 ÷ 6  = $0.0000233 / 回合
  Arc 重生：$0.000365 ÷ 70 = $0.0000052 / 回合

早期（≤12 回合）  $0.000767 + $0.0000285 = $0.000796 / 回合
中期（≤40 回合）  $0.000822 + $0.0000285 = $0.000851 / 回合  ← 計算基準
後期（40+ 回合）  $0.000863 + $0.0000285 = $0.000891 / 回合
```

> **計算基準採中期 $0.000851 / 回合**（最具代表性）

## 六、風險評估

### 極端使用率風險
若有用戶每日 100% 用完所有回合：
- 說書人 100% 使用：AI 成本 $4.60，收入 $18，Stripe 費 $0.82 → 淨利 $12.58（仍有 70%）
- **不存在虧損風險**，因定價已有充分緩衝

### Stripe 固定費影響
- 訂閱 $8：固定費 $0.30 佔 3.75%（可接受）
- 補充包 $4.99：固定費 $0.30 佔 6.0%（可接受）
- 任何低於 $3 的商品：固定費佔比超過 10%，**不建議**

### summary 成長上限
每 6 回合追加一段摘要（~50 字），約 40 個 token。
system prompt 輸入 token 從早期 ~2,623 緩慢增長至後期 ~2,945，**增加約 12%**，費用影響有限。

## 七、技術實作需求（Phase 1）

```
Supabase 表結構：
  user_subscriptions
    ├── user_id
    ├── plan: 'wanderer' | 'storyteller'
    ├── daily_limit: 60 | 180
    ├── turns_used_today: int
    ├── last_reset_date: date
    ├── stripe_subscription_id
    └── valid_until: timestamp

  user_addon_turns
    ├── user_id
    └── remaining_turns: int  （補充包，不過期）

每回合 server-side 扣款邏輯：
  1. 先扣 addon_turns（補充包優先或訂閱優先，可設定）
  2. 再扣 turns_used_today（檢查是否超過 daily_limit）
  3. 若均耗盡 → 返回 402 需要升級
```
